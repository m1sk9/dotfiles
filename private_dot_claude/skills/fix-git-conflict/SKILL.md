# タスク: Git コンフリクトの解消

以下の手順でコンフリクトを確認・解消してください．**プッシュは絶対に実行しないこと．**

## 1. 現在のブランチを確認

`git branch --show-current` で現在のブランチを確認してください．

- **main ブランチの場合**: 作業対象のブランチが正しいか，ユーザーに確認を取り，許可が得られなければ処理を中止する
- **main 以外のブランチの場合**: 次のステップに進む

## 2. PR の状態を確認

`gh pr view --json number,title,state,mergeable,mergeStateStatus` を実行し，現在のブランチに紐付く PR の状態を確認してください．

以下の点を確認し，問題がある場合はユーザーに報告して確認を取ること：
- PR が存在しない場合: 警告を出してユーザーに続行するか確認する
- `state` が `OPEN` 以外の場合（MERGED, CLOSED など）: ユーザーに確認する
- `mergeable` が `CONFLICTING` 以外で，コンフリクトが発生していない場合: その旨をユーザーに伝えて確認する

PR の情報（番号・タイトル・マージ可否）をユーザーに提示する．

## 3. main との差分を確認

以下を実行して，main との乖離状況を把握してください：

```
git fetch origin main
git log --oneline origin/main..HEAD
git log --oneline HEAD..origin/main
```

以下をユーザーに提示する：
- 現在のブランチが main より **先行しているコミット数**
- main が現在のブランチより **先行しているコミット数**（= rebase で取り込む変更数）

## 4. rebase の実行

`git rebase origin/main` を実行してください．

- rebase が正常に完了した場合: ステップ 6 に進む
- コンフリクトが発生した場合: ステップ 5 に進む

## 5. コンフリクトの解消

`git diff --diff-filter=U --name-only` でコンフリクトが発生しているファイルを特定してください．

各ファイルについて：

1. ファイルの内容を読み込み，`<<<<<<<`，`=======`，`>>>>>>>` マーカーを確認する
2. 変更の意図を理解した上で，適切な解決方針を判断する（両方を保持 / どちらかを採用 / 統合）
3. コンフリクトマーカーを取り除き，正しい内容に編集する
4. `git add <file>` でステージングする
5. すべてのコンフリクトを解消したら `git rebase --continue` を実行する

追加のコンフリクトが発生した場合は，同じ手順を繰り返す．

rebase に失敗して中断が必要な場合は，`git rebase --abort` を実行してユーザーに報告する．

## 6. 完了報告

以下をユーザーに提示してください：

- **解消したコンフリクトファイルの一覧**
- **各ファイルでの解決方針の説明**（どちらの変更を採用したか，なぜか）
- **現在のブランチ状態** (`git log --oneline -5`)
- **次のステップの案内**: 内容を確認したうえで `git push --force-with-lease` でプッシュしてください

> プッシュは事故防止のためユーザー自身が実行してください．このスキルでは絶対にプッシュを行いません．
