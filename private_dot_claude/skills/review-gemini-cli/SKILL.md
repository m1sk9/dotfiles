# タスク: Gemini CLI による PR コードレビュー

以下の手順で現在のブランチの PR をレビューし，結果を投稿してください．

## 1. PR 番号の取得

引数 `$ARGUMENTS` で PR 番号が指定されている場合はそれを使用してください．

指定がない場合は，以下のコマンドで現在のブランチに紐づく PR 番号を取得してください：

```sh
gh pr list --head (git branch --show-current) --json number --jq '.[0].number'
```

PR が見つからない場合は，その旨を報告して停止してください．

## 2. diff を Gemini CLI にレビューさせる

`git diff main...HEAD` の結果を Gemini CLI に渡してレビューさせ，結果を `/tmp/gemini_review.txt` に保存してください：

```sh
git diff main...HEAD | gemini -y --model gemini-3.1-pro-preview -p "以下のdiffをコードレビューしてください．バグ・セキュリティ・設計上の問題・改善点を具体的に指摘してください:\n$(cat)" > /tmp/gemini_review.txt
```

diff が空の場合は，レビュー対象がない旨を報告して停止してください．

## 3. レビュー結果を PR に投稿する

`/tmp/gemini_review.txt` の内容を以下のフォーマットで PR にレビューコメントとして投稿してください．`<PR番号>` は手順 1 で取得した番号に置き換えてください：

```sh
gh pr review <PR番号> --comment --body "$(cat <<'EOF'
🤖 AI Code Review

> Generated by: Gemini CLI (reviewer) + Claude Code (orchestrator)
> Model: gemini-3.1-pro-preview
> Scope: git diff main...HEAD

---

$(cat /tmp/gemini_review.txt)

---
このレビューは自動生成されたものです．判断は人間のレビュアーが行ってください．
EOF
)"
```

> [!NOTE]
> 外側の `<<'EOF'`（クォート付き）はシェルの履歴展開を防ぎますが，本文中の `$(cat /tmp/gemini_review.txt)` はコマンド置換で展開されレビュー結果が埋め込まれます．

## 4. 完了報告

投稿が完了したら，以下のコマンドで PR の URL を取得して表示してください：

```sh
gh pr view <PR番号> --json url --jq '.url'
```
