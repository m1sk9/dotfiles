# CLAUDE.md

## コードを変更する前に

- そのファイルを実際に読んだか？ 読んでいないコードを推測で変更してはいけない．
- 変更の影響範囲を把握しているか？ 呼び出し元・依存先を確認したか？
- この変更は自明か？ 非自明なら，方針（アプローチ・トレードオフ・手順）を提示して承認を得たか？
- 既存のコードスタイル・命名規則・コメント規約に合わせているか？ 迷ったらプロジェクトの CLAUDE.md を参照しろ．

## ファイル・依存を操作する前に

- ファイルの作成・削除は危険な操作か？ 危険でなければ確認不要．危険なら承認を得ろ．
- 依存パッケージを追加しようとしていないか？ `cargo add` 等を実行する前に必ず承認を得ろ．

## コマンドを実行する前に

- シェルは Fish 前提で書いているか？ Bash/Zsh 固有の構文を使っていないか？

## コマンドが失敗したとき

- 原因は明らかか？ 明らかなら報告した上で自分で修正して再試行しろ．不明なら止まって報告しろ．

## テストについて

- 変更に対応するテストを作成・更新したか？ テストを実行して通ることを確認したか？
- テストの実行方法やフレームワークはプロジェクトの CLAUDE.md に従え．

## Git コミットする前に

- Conventional Commits に従っているか？
- コミットメッセージはすべて英語で書いているか？

## 応答する前に

- 不明点・曖昧な点を残したまま進めようとしていないか？ 分からなければ質問しろ．
- 回答は日本語で書いているか？
