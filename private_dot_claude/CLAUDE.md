# ユーザー設定: CLAUDE.md

## Claude Code としての立ちふるまい

- 実際にファイルを読んでから変更すること．読んでいないコードを推測で変更しないこと．
- 変更の影響範囲を把握すること．呼び出し元・依存先を確認すること．
- 変更が非自明な場合は，方針（アプローチ・トレードオフ・手順）を提示して承認を得ること．
- 既存のコードスタイル・命名規則・コメント規約に合わせること．迷った場合はプロジェクトの CLAUDE.md を参照すること．
- ファイルの作成・削除が危険な操作かどうかを判断すること．危険な場合は承認を得ること．
- 依存パッケージを追加する前に必ず承認を得ること．
- シェルは Fish を前提に書くこと．Bash/Zsh 固有の構文を使わないこと．
- コマンドが失敗した際，原因が明らかな場合は報告の上で自己修正・再試行すること．不明な場合は停止して報告すること．
- 変更に対応するテストを作成・更新し，実行して通過を確認すること．
- テストの実行方法・フレームワークはプロジェクトの CLAUDE.md に従うこと．
- コミットメッセージは Conventional Commits に従い，すべて英語で書くこと．
- 作成するプルリクエストのタイトルは，そのまま CHANGELOG の一項目として使えるように書くこと．
- 不明点・曖昧な点が残っている場合は進めずに質問すること．
- 回答は日本語で書くこと．
- Git 系の操作を行った際に SSH 認証や GPG 鍵のエラーが出た場合は手動実行をユーザーに委ねること．その際実行すべきコマンドを示すこと．
- GitHub CLI を使って私 (@m1sk9) としてコメントや Issue 起票などの対外的なアクションを行う場合は，必ず「Generated by Claude Code」という文章を入れること．
- 2行以上に跨ぐコマンドはターミナルからのコピーがうまくいくように表示を工夫すること．

## 情報源の扱い

- Zenn / Qiita を情報源として使わないこと．対象ドメインは以下:
  - `qiita.com` / `*.qiita.com`
  - `zenn.dev` / `*.zenn.dev`
- プログラミングスクール・学習教材系のサイト（SEO / アフィリエイト目的の解説記事を含む）も同様に情報源として使わないこと．
- 上記ドメイン・カテゴリについては，ページを開かない・記事を要約しない・引用しない・内容を根拠や実装方針に使わないこと．
- 検索結果に Qiita / Zenn / プログラミングスクール系サイトが出てきた場合は無視し，別の情報源を探すこと．
- Web 検索時は可能な限り以下の除外条件を付けること: `-site:qiita.com -site:zenn.dev`
- 優先する情報源は以下の順とする:
  1. 公式ドキュメント
  2. 公式 GitHub リポジトリ
  3. RFC / 仕様書 / 標準規格
  4. 公式 Issue / Discussion
  5. メンテナのブログやリリースノート
  6. 信頼できる企業の技術ブログ
- Stack Overflow は実務的なトラブルシュート時のみ補助的に使うこと．
- Qiita / Zenn なしで十分な根拠が見つからない場合は，Qiita / Zenn を使わずに「一次情報または信頼できる情報源が見つからない」と明示すること．
