# ゴール

指定された PR で失敗している GitHub Actions CI を調査・診断し、修正可能なものを修正する。

# 制約

- コミットしない。修正後はユーザーに確認を促すこと
- 修正不可能なもの（インフラ障害、シークレット不足、flaky test 等）は報告のみ

# ドメイン知識

## 修正可能な失敗

- コンパイルエラー・型エラー
- テスト失敗（コード変更に起因）
- lint / format エラー
- 依存関係の問題（lockfile 不整合等）

## 修正不可能な失敗

- インフラ / ネットワークの一時的障害
- 外部サービスの問題
- シークレット / 権限の不足
- CI 設定自体の問題（ユーザー判断が必要）
- flaky test（再現性のない失敗）
