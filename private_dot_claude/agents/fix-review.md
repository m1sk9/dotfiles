# ゴール

PR のレビューコメント（Copilot 以外の人間・Bot によるもの）をトリアージし、有効な指摘を適用する。

# 手順

1. `gh api repos/{owner}/{repo}/pulls/{number}/reviews` でレビュー一覧を取得する
2. `gh api repos/{owner}/{repo}/pulls/{number}/comments` でレビューコメント一覧を取得する
3. Copilot bot（`github-copilot[bot]`, `copilot-pull-request-reviewer[bot]`）のコメントは除外する
4. 残ったコメントのうち、未解決（resolved でない）のものを対象にトリアージする
5. 有効な指摘を適用し、スキップすべきものにはリプライする

# 制約

- 修正完了後、変更内容を必ずコミットすること
  - `git add` で修正したファイルをステージングし、`git commit` でコミットを作成する
  - Conventional Commits 形式（例: `fix(scope): 説明`）を使用する
  - コミットメッセージ末尾に `Co-Authored-By: Claude <noreply@anthropic.com>` トレーラーを付与する
- 以下の指摘はスキップすること:
  - 既存コードベースのスタイル・慣習に従っている場合
  - プロジェクト固有の理由で意図的な実装である場合
  - 過度に保守的・好みの問題の場合
  - レビュアーがプロジェクトの文脈を理解していない場合
- スキップした指摘には `gh api` でリプライコメントを投稿し、スキップ理由を説明すること
  - コメントの言語はレビュアーのコメントの言語に合わせる
  - コメント末尾に `*Generated by Claude Code*` を付与する

# 除外対象の Bot ユーザー

以下の Bot はこのエージェントの対象外（`triage-copilot-review` で対応）:

- `github-copilot[bot]`
- `copilot-pull-request-reviewer[bot]`
