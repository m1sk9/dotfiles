# ゴール

稼働中の Hunk セッションに**ユーザーが書いた inline コメント**（`--type user`）を読み、妥当な指摘をローカルの作業ツリーに適用する。指摘のソースが GitHub PR ではなくローカルの Hunk である点が `fix-review` との違い。

引数: 対象を絞るファイルパス（任意。省略時はセッション全体）。

# 手順

## 1. セッションとコメントを取得する

1. `hunk session list --json` で live セッションを確認する。現在のリポジトリ（`repoRoot`）に一致するものが無ければ、**適用対象が無い**旨を報告し、ユーザーに `hunk diff` の起動とコメント追加を促して終了する（このスキルはコメントの存在が前提なので、ここは例外的にユーザーへ起動を促してよい）。
2. `hunk session comment list --repo . --type user --json` でユーザーコメントを取得する（引数でファイル指定があれば `--file <path>` を付ける）。各コメントの **id・filePath・行・本文**を控える。
3. コメントが 0 件なら、その旨を報告して終了する。

## 2. トリアージして適用する

- 各コメントを指摘として解釈し、妥当なものを Edit で適用する。
- 対象ファイルを Read し、周辺の既存スタイル・命名・慣習に合わせること。
- 指示が曖昧・複数解釈できる場合は、勝手に決めず方針を提示してユーザーに確認すること。

## 3. 対応結果を Hunk に反映する

- **適用したコメント**: 元のユーザーコメントを `hunk session comment rm --repo . <comment-id>` で削除して「対応済み」を示す。**id は必ず手順 1 の `--json` 出力の `id` フィールドから取る**（`comment list` のテキスト表示の見かけ上の id とは異なることがある）。
- **スキップしたコメント**: ユーザーコメントは残し、`hunk session comment add` で理由を添えた agent note を付ける（TUI 上でユーザーがスキップ理由を確認できる）。
- 変更後に `hunk session reload --repo . -- diff` で diff を更新し、ユーザーが最新状態を見られるようにする。

## 4. コミットしない

- このエージェントはローカルの対話的レビュー用であり、**コミット・プッシュはしない**。適用後の確認とコミットはユーザーに委ねる（必要なら `commit` スキルを使う）。

# 制約

- ユーザーの指摘であっても、既存コードベースのスタイルや意図的な実装に反する・明らかに壊す変更は、そのまま適用せず確認すること。
- `hunk diff` / `hunk show` を直接叩かず、必ず `hunk session *` を使うこと。
- CLI の詳細仕様は `hunk-review` スキルを参照。

# 返り値

- 読み取ったユーザーコメント件数
- 適用した指摘の概要（ファイル・変更内容）
- スキップした指摘と理由
- ユーザーの判断が必要な残件（あれば）
